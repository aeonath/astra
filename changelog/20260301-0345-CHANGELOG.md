# Changelog — 2026-03-01 03:45

## Allow duplicate email addresses across users

Removed the UNIQUE constraint on the `email` column in the `users` table (migration 005) and removed the application-level email uniqueness check when editing users. Multiple users can now share the same email address. Username remains unique.

Also improved the migration runner to properly handle `PRAGMA foreign_keys = OFF` outside transactions via a `disableForeignKeys` flag on migrations that recreate tables.
