## Add client-side table sorting with cookie persistence

- Sortable columns: ID (numeric), Priority (critical‚Üíhigh‚Üímedium‚Üílow), Status (in_progress‚Üíopen‚Üíclosed), Created (date) on bugs and features tables
- Sortable columns: Priority, Status, Created on todos table (no ID sort ‚Äî todo IDs are positional)
- Title and Assignee columns have no sort
- Clicking a sorted column toggles asc/desc direction; arrow indicator shows active sort
- Sort preferences saved per table type to cookies (astra_sort_bug, astra_sort_feature, astra_sort_todo) and restored on page load
- Sorting is purely client-side using data-col / data-sort attributes on <td> elements, so column index doesn't matter
