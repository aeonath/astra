# Changelog — 2026-03-06 15:17

## Advanced search with filters

- GET /bugs/search: rebuilt with filter params — q, type, status, priority, project, assignee (any/unassigned/user), date_from, date_to
- Dynamic SQL query builder: conditions and params arrays assembled from whichever filters are active
- hasAnyFilter flag: when no params at all, shows the Advanced Search form with welcome text instead of results
- Empty navbar search submit naturally lands on advanced search page (no JS required)
- Active filter summary bar: shows result count + pill tags for each active filter
- Clear filters link appears when any filter is active
- Results table: up to 200, critical rows bold red, "Feature Request" label fixed, clickable rows
- CSS: .advanced-search-card, .search-row grid layout (4 cols), .search-filter-tag pills, .search-welcome
