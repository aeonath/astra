# Changelog — 2026-03-06 15:15

## Push notifications for new public submissions via SSE

- src/notifications.js: in-memory SSE client registry with addClient, removeClient, broadcast
- admin.js: GET /admin/notifications/stream SSE endpoint with 25s heartbeat and auto-cleanup on disconnect
- projects.js: broadcasts new-submission event (type, title, submitter, project name) after saving submission
- admin-notifications.js: client-side EventSource that connects to the stream, shows a red badge count on the Submissions nav link, and fires a browser Notification (requests permission on first event); auto-reconnects on disconnect
- header.ejs: loads admin-notifications.js for canManageSubmissions users; adds #submissions-badge span inside Submissions link
- style.css: .submissions-badge — small red pill badge; auto-reconnect after 5s on connection drop
