# Changelog — 2026-03-02 10:00

## Fix: Quoted URLs in landing page project links

**Problem:** Clicking a project card on the landing page produced a URL with literal quote characters (e.g., `"/projects/lyric"` instead of `/projects/lyric`), resulting in a 404 error.

**Cause:** The landing page template (`landing.ejs`) used `<%= %>` (HTML-escaped output) to dynamically build `<a>` tags with href attributes. EJS escaped the `"` characters in the href to `&quot;`, which the browser then decoded as literal quotes embedded in the URL path.

**Fix:** Changed the four affected EJS tags from `<%= %>` to `<%- %>` (unescaped output) on lines 20, 41, 52, and 73. This is safe because the values are constructed from known-safe tag names and project slugs (which are auto-generated with no special characters).
