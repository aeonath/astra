# Changelog — 2026-02-28 21:30

## Add public/private visibility control for projects

- Added `public` column to the projects table (defaults to private)
- Built a migration system in `src/db/schema.js` using a `_migrations` table so existing databases get the new column automatically
- Admin can toggle project visibility (Public/Private) from the admin projects page
- Admin create project form now includes a "Publicly visible" checkbox (checked by default)
- Landing page and `/projects` list only show public projects to unauthenticated visitors; logged-in users see all active projects
- Private projects return "not found" for unauthenticated visitors on `/projects/:slug`

## Add CLAUDE.md and changelog workflow

- Created `CLAUDE.md` with project architecture, commands, conventions, and workflow rules
- Established changelog convention: every change gets a `changelog/YYYYMMDD-HHMM-CHANGELOG.md` entry
- Established commit convention: `git add .` and `git commit` after every task
