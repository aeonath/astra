- Bug/feature creation form adapts labels and placeholders based on type
- Bug detail view shows a type badge (Bug/Feature) in the header and sidebar
- Breadcrumb links back to the correct view tab (bugs or features)
- Landing page shows bug breakdown stats plus feature request counts per project
- Project index cards show open bug and open feature counts
- Added purple-themed CSS for feature buttons, badges, and stats
- Feature visibility follows the existing project `public` flag — no separate toggle needed
