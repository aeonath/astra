# Changelog — 2026-03-01 02:45

## User management improvements

- **Edit users:** Clicking a user row in the admin user table now populates the form for editing their display name, email, and role. Username is read-only after creation. Password field is optional when editing (leave blank to keep current).
- **Admin accounts cannot be disabled:** The Disable button is hidden for admin users in the UI, and the server-side route rejects the request with an error.
- **Removed "CLI only" label** from admin user rows — it had no useful context.
- **Password reset** and **disable/enable** buttons only appear for non-admin users.
