## 2026-03-11 — Add internally visible / publicly visible project controls

- Public view now only shows project name and open bug/FR counts (no description or details)
- Added `internally_visible` column to projects (migration 020)
- Logged-in users only see projects where `internally_visible = 1`
- Public visitors only see projects where `public = 1`
- Admin projects form now has two checkboxes:
  - "Internally visible" — controls if project appears for logged-in users
  - "Publicly visible" — only available when internally visible is checked, controls public view
- Checking "Internally visible" defaults "Publicly visible" to checked
- Unchecking "Internally visible" hides and unchecks "Publicly visible"
- If not internally visible, `public` is forced to 0 server-side
